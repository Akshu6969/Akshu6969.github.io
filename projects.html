Microcontrolled Video Game Integration

ECE 2035 Final Project

For my final project in Programming for HW/SW systems, I developed a custom, hardware-oriented video game utilizing an ESP32 microcontroller and physical inputs. The project involved writing complex C script routines to handle physical game inputs and translate them into on-screen actions on an external display. The core logic required managing animated sprites, developing strict collision rules, and engineering real-time weapon mechanics. Ensuring the game loop ran efficiently without frame drops necessitated extensive debugging of runtime application crashes and logic errors.

Built using: ESP32 Microcontroller, C, external display, physical pushbuttons.

Slay the Spire 2 "Chaos Mods"

ECE 1100 Individual Discovery Project

For my ECE Discovery Studio course, I focused on modifiying game code and card mechanics. I conceptualized and programmed a custom expansion for the game Slay the Spire 2. Utilizing C# and the HarmonyLib patching library, I engineered a mod that intercepts and alters structural variables at runtime. Specifically, the script targets and replaces MapPointType.Shop with customized chaotic logic, while rigorously ensuring that all other standard map points are preserved. The integration successfully targets MapPointType.Unknown events to inject new mechanics without disrupting the game's underlying stability.

Built using: C#, HarmonyLib, Rider.
